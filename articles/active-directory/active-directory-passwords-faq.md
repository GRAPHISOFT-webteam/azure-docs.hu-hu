# 密碼管理常見問題集

以下是與密碼管理相關之所有事項的一些常見問題。

如果您發現自己不知道答案的問題，或者要對所面臨到的問題尋求協助，您可以參閱下方資訊以查看我們是否已涵蓋該問題。 如果我們尚未涵蓋該問題，不用擔心！ 提出的疑問，不在這裡討論 [Azure AD 論壇](https://social.msdn.microsoft.com/Forums/home?forum=WindowsAzureAD) ，我們會得到給您，我們可以。

## 密碼重設註冊

 - **問: 我的使用者註冊自己的密碼重設資料嗎?**

> **A:** 是，只要已啟用密碼重設並且獲得授權，他們可以移至 http://aka.ms/ssprsetup 註冊要用於密碼重設的驗證資訊的密碼重設註冊入口網站。 使用者也可以登錄 http://myapps.microsoft.com 的存取面板，按一下 [設定檔] 索引標籤，然後按一下 [註冊密碼重設] 選項。 深入了解如何讓您的使用者設定密碼重設，方法是閱讀如何讓使用者設定重設密碼。

 - **問: 是否可以代表我的使用者定義密碼重設資料?**

> **A:** 是，您可以使用 DirSync 或 PowerShell，或者透過 [Azure 管理入口網站](https://manage.windowsazure.com) 或 Office Admin 入口網站。 藉由部落格文章「Azure AD MFA 和密碼重設電話號碼的改進隱私權」和閱讀「了解密碼重設如何使用資料」，來深入了解此功能。

 - **問: 是否可以同步處理資料從內部部署的安全性問題?**

> **A:** 否，目前還不可能，但是我們會加以考量。

 - **問: 是否可以我的使用者可以註冊資料的方式，其他使用者無法看見此資料?**

> **A:** 是的當使用者註冊使用密碼重設註冊入口網站會儲存到私人驗證欄位，只有全域管理員和使用者自己看得到的資料。 藉由部落格文章「Azure AD MFA 和密碼重設電話號碼的改進隱私權」和閱讀「了解密碼重設如何使用資料」，來深入了解此功能。

 - **問: 我的使用者必須註冊才能使用密碼重設?**

> **A:** 否，如果您定義足夠的驗證資訊代表他們執行時，使用者將不需要註冊。 只要您已正確地格式化儲存在目錄中適當欄位的資料，密碼重設功能就會順利運作。 藉由閱讀「了解密碼重設如何使用資料」以深入了解。

 - **問: 可以同步處理或代表我的使用者設定的驗證電話、 驗證電子郵件或備用驗證電話欄位?**

> **A:** 目前為否，但是我們考慮啟用這項功能。

 - **問: 如何註冊入口網站會得知向使用者顯示那些選項?**

> **A:** 密碼重設註冊入口網站只顯示的選項，您已啟用目錄的 [設定] 索引標籤的使用者密碼重設原則區段下，為使用者。 這表示如果您未啟用，例如安全性問題，則使用者無法註冊該選項。

 - **問: 何時會在使用者被視為已註冊?**

> **A:** 使用者才算完成註冊他或她至少有 N 個驗證資訊定義，其中 N 是數字的方法需要驗證您已設定在 [Azure 管理入口網站](https://manage.windowsazure.com)。 若要深入了解，請參閱「自訂使用者密碼重設原則」。


## 密碼重設

 - **問: 如何長時間，我應該等待以接收密碼重設的電子郵件、 SMS 或電話?**

> **A:** 電子郵件、 SMS 訊息和電話應該在 1 分鐘之內，送達一般情況下為 5-20 秒。 如果您在此時間範圍內沒有收到通知，請檢查垃圾郵件資料夾中連絡的號碼/電子郵件是您預期的，且目錄中的驗證資料已正確格式化。 若要深入了解搭配格式化電話號碼和電子郵件地址使用密碼重設，請參閱「了解密碼重設如何使用資料」。

 - **問: 哪些語言支援密碼重設?**

> **A:** 密碼重設 UI、 SMS 訊息以及語音通話都以 Office 365 中支援的相同 40 個語言當地語系化。 這些語言為：阿拉伯文、保加利亞文、簡體中文、繁體中文、克羅埃西亞文、捷克文、丹麥文、荷蘭文、英文、愛沙尼亞文、芬蘭文、法文、德文、希臘文、希伯來文、印度文、匈牙利文、印尼文、義大利文、日文、哈薩克文、韓文、拉脫維亞文、立陶宛文、馬來文 (馬來西亞)、挪威文 (巴克摩)、波蘭文、葡萄牙文 (巴西)、葡萄牙文 (葡萄牙)、羅馬尼亞文、俄文、塞爾維亞文 (拉丁)、斯洛伐克文、斯洛維尼亞文、西班牙文、瑞典文、泰文、土耳其文、烏克蘭文和越南文。

 - **問: 密碼重設體驗的哪些部分會有商標時設定組織商標在我的目錄中的 [設定] 索引標籤?**

> **A:** 密碼重設入口網站會顯示您的組織商標，也會讓您設定 「 連絡您的系統管理員連結以指向自訂電子郵件或 URL。 密碼重設所傳送的任何電子郵件都會包含您的組織商標、顏色 (此案例為紅色)、電子郵件內文中的名稱，以及自訂的寄件者名稱。 請參閱下方具有所有商標項目的範例。 若要深入了解，請閱讀「自訂密碼重設外觀」。

  ![][001]

 - **問: 如何教育使用者去何處重設其密碼?**

> **A:** 您可以傳送至 https://passwordreset.microsoftonline.com 使用者直接，或者可以指示他們按一下無法存取任何學校或工作 ID 的登入畫面上找到您的帳戶 」 連結。 您可以放心地在使用者可以輕易存取的任何位置發佈這些連結 (或建立 URL 以重新導向至它們)。

 - **問: 是否我可以使用此頁面上，從行動裝置?**

> **A:** 是，此頁面可以在行動裝置上運作。

 - **問: 支援解除鎖定本機 active directory 帳戶的使用者重設其密碼時?**

> **A:** 是，使用者當重設其密碼且密碼回寫已部署於所有版本的 Azure AD Connect 或 Azure AD sync 1.0.0485.0222 版或更新版本，則該使用者帳戶時，會自動解除鎖定該使用者重設其密碼。

 - **問: 如何整合密碼重設直接到我的使用者的桌面登入體驗?**

> **A:** 目前還不可能。 不過，如果您絕對需要這項功能，而且是 Azure AD Premium 客戶，您不需要額外成本即可安裝 Microsoft Identity Manager，並且部署內部部署密碼重設方案，以解決這項需求。

 - **問: 是否可以設定不同的地區設定不同的安全性問題?**

> **A:** 否，目前還不可能，但是我們會加以考量。

 - **問: 如何許多問題我們可以設定的安全性問題驗證選項?**

> **A:** 您可以設定多達 20 個自訂安全性問題在 [Azure 管理入口網站](https://manage.windowsazure.com)。

 - **問: 保留時間長度可能安全性問題?**

> **A:** 安全性問題可以介於 3 到 200 個字元之間。

 - **問: 如何長安全性問題的答案可能?**

> **A:** 答案可能是 3 到 40 個字元。

 - **問: 有重複答案已拒絕的安全性問題?**

> **A:** 是，我們會拒絕重複回答安全性問題。

 - **問: 使用者是否可以註冊一個以上的相同安全性問題?**

> **A:** 否，一旦使用者註冊特定的問題，他或她不可以註冊該問題第二次。

 - **問: 是否可以設定註冊的安全性問題的最小限制和重設?**

> **A:** 是的設定一項限制，以及重設註冊。 註冊可能需要 3-5 個安全性問題，重設也需要 3-5 個安全性問題。

 - **問: 如果使用者已註冊超過重設所需的問題數目上限，如何選取安全性問題重設期間?**

> **A:** 使用者已註冊的其中 N 是所需的密碼重設問題的最小數目問題總數當中隨機選取 n 個安全性問題。 例如，如果使用者已註冊 5 個安全性問題，但是重設只需要 3 個，則會在 5 個問題當中隨機選取 3 個，並且在重設時對使用者顯示。 如果使用者回答問題的答案錯誤，則選取程序會重新發生以避免問題 hammering。

 - **您問防止使用者嘗試密碼重設在短時間內多次嗎?**

> **A:** 是，有幾項密碼重設內建的安全性功能。 使用者在一個小時之內只能嘗試 5 次密碼重設，否則會鎖定 24 小時。 使用者在一個小時之內只能嘗試驗證電話號碼 5 次，否則會鎖定 24 小時。 使用者在一個小時之內只能嘗試單一驗證方法 5 次，否則會鎖定 24 小時。

 - **問: 時間長度是電子郵件和 SMS 單次密碼有效?**

> **A:** 密碼重設工作階段存留期為 105 分鐘。 這表示從密碼重設作業開始，使用者有 105 分鐘可以重設其密碼。 這段時間之後，電子郵件和 SMS 單次密碼就會無效。


## 密碼管理報告

 - **問: 如何花費時間顯示在密碼管理報告上的資料?**

> **A:** 資料應該在密碼管理報告上會顯示在 5-10 分鐘內。 某些情況下，可能會花費多達一小時才顯示。

 - **問: 如何可以篩選密碼管理報告?**

> **A:** 您就可以按一下最右邊資料行標籤報表頂端的小放大鏡來篩選密碼管理報告 (請參閱螢幕擷取畫面)。 如果您想要做更豐富的篩選，您可以將報告下載到 excel，並建立樞紐分析表。

  ![][002]

 - **問：儲存於密碼管理報表中的事件上限數是？**

> **A:** 最多 1000 密碼重設或密碼重設註冊事件會儲存在密碼管理報告。 我們正在擴展此數目，以包含更多的事件。

 - **問: 多遠密碼管理報告可以回溯嗎?**

> **A:** 密碼管理報告會顯示過去 30 天內發生的作業。 我們目前正在調查要如何讓這一段時間更長。 現在，如果您需要封存這項資料，您可以定期下載報告並將它們儲存在不同的位置。

 - **問: 是否有密碼管理報告可以顯示的資料列的數目上限?**

> **A:** 是，最多 1000 個資料列可能會出現在任何一種密碼管理報表中，是否就顯示在 UI 中或是下載。 我們目前正在調查要如何增加此限制。

 - **問: 有 API 來存取密碼重設或報告資料的註冊嗎?**

> **A:** 是，請參閱下列文件，若要了解如何存取報告的資料流的重設密碼。 [了解如何存取密碼重設，以程式設計的方式報告事件](https://msdn.microsoft.com/library/azure/mt126081.aspx#BKMK_SsprActivityEvent)。

## 密碼回寫

 - **問: 密碼回寫如何在幕後運作?**

> **A:** 看到密碼回寫如何運作的詳細說明當您啟用密碼回寫，以及如何資料透過系統回流到您的內部部署環境時，會發生什麼事。 請參閱「密碼回寫如何運作」中的「密碼回寫安全性模型」，以了解我們如何確保密碼回寫是高度安全的服務。

 - * * 多久問: 密碼回寫攜帶至運作? 是否有像是使用密碼雜湊同步處理同步處理延遲? * *

> **A:** 密碼回寫是即時。 它是同步管線，基本上的運作與密碼雜湊同步處理不同。 密碼回寫可以讓使用者即時取得密碼重設或變更作業成功的回應。 成功回寫密碼的平均時間低於 500 毫秒。

 - **問: 哪些類型的帳戶密碼回寫運作的?**

> **A:** 密碼回寫適用於同盟和密碼雜湊同步的使用者。

 - **問: 密碼回寫強制執行我的網域密碼原則?**

> **A:** 是，密碼回寫會強制執行密碼使用期限、 歷程記錄、 複雜度、 篩選和其他您可能會放在您的本機網域中密碼上的限制。

 - * * 問: 是安全的密碼回寫? 我要如何確定我將不會被駭客破解? * *

> **A:** ，密碼回寫是非常安全。 若要進一步了解密碼回寫服務實作的 4 個層級的安全性，請參閱「密碼回寫如何運作」中的「密碼回寫安全性模型」。
