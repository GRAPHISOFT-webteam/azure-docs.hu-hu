# <a name="azure-active-directory-identity-protection-faq"></a>Az Azure Active Directory Identity Protection – gyakori kérdések


## <a name="why-do-some-risk-events-have-closed-system-status"></a>Miért van néhány kockázati események "(rendszer) Lezárva" állapota?

**V:** ezek olyan eseményeket, amelyek az Azure Active Directory Identity Protection észlelt, és később bezárva, mert az események nem vették kockázatos. Ezek az események nem számítanak bele a felhasználó kockázati szintjét. 

---

## <a name="do-i-need-to-be-a-global-admin-to-use-identity-protection-in-the-azure-portal"></a>Van-e globális rendszergazda az Azure portálon Identity Protection használatához?
**V:** **nem**. Meg lehet egy biztonsági olvasó, a biztonsági rendszergazda vagy egy globális rendszergazdai Identity Protection használatához.

**V:** lásd [Ismerkedés az Azure Active Directory Premium](active-directory-get-started-premium.md) a választ a kérdésére.
