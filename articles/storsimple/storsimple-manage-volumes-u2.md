# 使用 StorSimple Manager 服務來管理磁碟區

[AZURE.INCLUDE [storsimple-version-selector-manage-volumes](../../includes/storsimple-version-selector-manage-volumes.md)]

## 概觀

本教學課程說明如何使用 StorSimple Manager 服務來建立和管理已安裝 Update 2 的 StorSimple 裝置與 StorSimple 虛擬裝置上的磁碟區。

StorSimple Manager 服務是 Azure 入口網站的延伸模組，可讓您透過單一 Web 介面管理 StorSimple 解決方案。 除了管理磁碟區，您可以使用 StorSimple Manager 服務來建立和管理 StorSimple 服務、檢視和管理裝置、檢視警示，以及檢視和管理備份原則與備份類別目錄。

## 磁碟區類型

StorSimple 磁碟區可以是：

- **在本機固定磁碟區**: 隨時都在這些磁碟區中的資料會保留在本機的 StorSimple 裝置。
- **階層的磁碟區**: 這些磁碟區中的資料可以散至雲端。

封存的磁碟區是一種分層磁碟區。 用於封存磁碟區的較大重複資料刪除區塊大小，可讓裝置將資料的較大區段傳輸至雲端。 

如果有需要，您可以將磁碟區類型從本機變更為分層或從分層變更為本機。 如需詳細資訊，請移至 [變更磁碟區類型](#change-the-volume-type)。

### 固定在本機的磁碟區

固定在本機的磁碟區是完整佈建的磁碟區，不會將資料分層至雲端，因此可以確保主要資料在本機當中，獨立於雲端連線之外。 固定在本機的磁碟區的資料不會進行重複資料刪除和壓縮，但是，固定在本機的磁碟區的快照會進行重複資料刪除。 

固定在本機的磁碟區是完整佈建，因此，當您建立它們時，必須在裝置上有足夠的空間。 您可以在 StorSimple 8100 裝置上佈建多達 9 TB 大小的固定在本機的磁碟區，在 8600 裝置上佈建多達 24 TB 的固定在本機的磁碟區。 StorSimple 會為快照、中繼資料和資料處理在裝置上保留剩餘的本機空間。 您可以將固定在本機的磁碟區大小增加到可用的最大空間，但是建立之後就不能減少磁碟區的大小。

當您建立固定在本機的磁碟區時，建立分層磁碟區的可用空間會減少。 反之亦然：如果您有現有的分層磁碟區，可用於建立固定在本機的磁碟區的空間將會低於上述的最大限制。

### 分層磁碟區

分層磁碟區是精簡佈建磁碟區，經常存取裝置上本機的資料，而較少使用的資料會自動分層至雲端。 精簡佈建是一種虛擬化技術，精簡佈建中的可用儲存體會顯示超過實體資源。 與其預先保留足夠的儲存空間，StorSimple 會使用精簡佈建來配置剛好符合目前需求的足夠空間。 雲端儲存體的彈性本質正好支援這種方法，因為 StorSimple 可以增加或減少雲端儲存體以符合不斷變更的需求。

您可以指定分層式磁碟區做為保存的磁碟區藉由檢查 **用於較不常存取的封存資料的這個磁碟區** 核取方塊。 封存磁碟區使用 512 KB 重複資料刪除區塊大小，而非封存磁碟區使用 64 KB 區塊大小。 較大的重複資料刪除區塊大小可讓裝置將資料的較大區塊傳輸至雲端。

>[AZURE.NOTE] 保存 StorSimple 的更新前 2 版本所建立的磁碟區將會匯入為階層以保存選取的核取方塊。

## [磁碟區] 頁面

 **磁碟區** 頁面可讓您管理為啟動器 (伺服器) 的 Microsoft Azure StorSimple 裝置佈建的存放磁碟區。 它會顯示 StorSimple 裝置上的磁碟區清單。

 ![磁碟區頁面](./media/storsimple-manage-volumes-u2/VolumePage.png)

磁碟區是由一系列屬性所組成：

- **磁碟區名稱** – 必須是唯一且有助於識別磁碟區的描述性名稱。 這個名稱也可在您篩選特定磁碟區時用於監視報告。

- **狀態** – 可以是線上或離線。 如果是離線的磁碟區，允許存取使用它的啟動器 (伺服器) 會看不到該磁碟區。

- **容量** – 指定啟動器 (伺服器) 可以儲存的資料總量。 固定在本機的磁碟區是完整佈建，而且位於 StorSimple 裝置上。 分層磁碟區是精簡佈建，而且資料會進行重複資料刪除。 使用精簡佈建磁碟區，您的裝置不會根據設定的磁碟區容量，在內部或在雲端預先配置實體儲存體容量。 磁碟區容量會依需求來配置和耗用。

- **型別** – 指出磁碟區是否 **階層** (預設值) 或 **在本機固定**。

- **備份** – 指出磁碟區是否有預設的備份原則。

- **存取** – 指定允許存取此磁碟區的啟動器 (伺服器)。 啟動器不是與磁碟區相關聯的存取控制記錄 (ACR) 成員，就看不到磁碟區。

- **監視** – 指定是否正在監視磁碟區。 預設會在建立磁碟區時啟用監視。 不過，磁碟區複製會停用監視。 若要啟用監視磁碟區，請依照下列中的指示 [監視磁碟區](#monitor-a-volume)。

## 新增磁碟區

您 [建立磁碟區](storsimple-deployment-walkthrough-u2.md#step-6-create-a-volume) 在部署 StorSimple 方案期間。 新增磁碟區會是類似的程序。

#### 若要新增磁碟區

1. 在 **裝置** 頁面中，選取裝置，並按兩下，然後按一下 **磁碟區容器** ] 索引標籤。

2. 從清單中選取磁碟區容器，然後按兩下它以存取與該容器相關聯的磁碟區。

3. 按一下 [ **新增** 頁面的底部。 [新增磁碟區精靈] 隨即啟動。

     ![加入磁碟區精靈基本設定](./media/storsimple-manage-volumes-u2/AddVolume1.png)

4. 在 [新增磁碟區精靈] 中，在 **基本設定**, ，執行下列動作:

  1. 提供 **名稱** 磁碟區。
  2. 選取 **使用類型** 從下拉式清單中。 需要在任何時間都在本機裝置上的可用資料的工作負載，請選取 **在本機固定**。 對於所有其他資料類型，選取 **階層**。 (**階層** 是預設值。)
  3. 如果您選取 **階層** 在步驟 2 中，您可以選取 **用於較不常存取的封存資料的這個磁碟區** 設定封存的磁碟區] 核取方塊。
  4. 輸入 **佈建的容量** GB 或 TB 磁碟區。 請參閱 [佈建的容量](#provisioned-capacity) 每個裝置與磁碟區類型的最大大小。 看看 **可用容量** 來判斷您的裝置上實際可以使用多少儲存空間。

    If you are configuring a locally pinned volume, you will see the following message.

    ![Change Volume type message](./media/storsimple-manage-volumes-u2/LocalVolMessage.png)

    The following page appears.

    ![Add Volume wizard Additional Settings](./media/storsimple-manage-volumes-u2/AddVolume2.png)<br>
   
5. 按一下箭頭圖示 ![箭號圖示](./media/storsimple-manage-volumes-u2/HCS_ArrowIcon.png)前往 **其他設定** 頁面。

6. 在 **其他設定**, ，加入新的存取控制記錄 (ACR):
  
  1. 在下拉式清單中選取存取控制記錄 (ACR)。 或者，您可以加入新的 ACR。 ACR 會藉由比對主機與記錄中列出的 IQN 來決定哪些主機可以存取磁碟區。 如果您尚未指定 ACR，您會看到下列訊息。

        ![指定 ACR](./media/storsimple-manage-volumes-u2/SpecifyACR.png)


  2. 我們建議您選取 **啟用此磁碟區的預設備份** 核取方塊。
  3. 按一下核取圖示 ![核取圖示](./media/storsimple-manage-volumes-u2/HCS_CheckIcon.png) 使用指定的設定建立磁碟區。

新的磁碟區現在已備妥可供使用。

>[AZURE.NOTE] 如果您建立在本機固定磁碟區，然後建立另一個在本機固定磁碟區立即之後，依序執行工作的建立磁碟區。 必須先完成第一個磁碟區建立作業，才能開始下一個磁碟區建立作業。

## 修改磁碟區

當您需要擴充磁碟區，或變更存取該磁碟區的主機時，請修改磁碟區。

> [AZURE.IMPORTANT] 
>
> - 如果您修改裝置上的磁碟區大小，也必須變更主機上的磁碟區大小。 
> - 此處所述的主機端步驟適用於 Windows Server 2012 (2012R2)。 Linux 或其他主機作業系統的程序會有所不同。 如果要在執行其他作業系統的主機上修改磁碟區，請參考主機作業系統的指示。 

#### 若要修改磁碟區

1. 在 **裝置** 頁面中，選取裝置，並按兩下，然後按一下 **磁碟區容器** ] 索引標籤。

2. 從清單中選取磁碟區容器，然後按兩下它以檢視與該容器相關聯的磁碟區。

3. 選取磁碟區，然後在頁面底部，按一下 [ **修改**。 [修改磁碟區精靈] 隨即啟動。

4. 在 [修改磁碟區精靈] 中，在 **基本設定**, ，您可以執行下列:

  - 編輯 **名稱**。
  - 轉換 **使用類型** 從本機釘選到階層，或從分層至本機固定 (請參閱 [變更磁碟區類型](#change-the-volume-type) 如需詳細資訊)。
  - 增加 **佈建的容量**。  **佈建的容量** 只能增加。 您無法在磁碟區建立後予以壓縮。

5. 在 **其他設定**, ，您可以修改 ACR，前提是磁碟區已離線。 如果磁碟區已連線，您必須先讓它離線。 中的步驟，請參閱 [使磁碟區離線](#take-a-volume-offline) 修改 ACR 之前。

    > [AZURE.NOTE] 您無法變更 **啟用預設備份** 磁碟區的選項。

6. 按一下核取圖示以儲存您的變更 ![核取圖示](./media/storsimple-manage-volumes-u2/HCS_CheckIcon.png). Azure 入口網站將會顯示更新磁碟區訊息。 如果磁碟區已成功更新，即會顯示成功訊息。

7. 如果您要延伸磁碟區，請在 Windows 主機電腦上完成下列步驟：

   1. 移至 **電腦管理** ]-> [**磁碟管理**。
   2. 以滑鼠右鍵按一下 **磁碟管理** ，然後選取 **重新掃描磁碟**。
   3. 在磁碟清單中，選取您要更新的磁碟區、 按一下滑鼠右鍵，然後選取 **延伸磁碟區**。 [延伸磁碟區精靈] 隨即啟動。 按一下 [ **下一步**。
   4. 使用預設值完成精靈。 完成精靈後，磁碟區應該會顯示增加的大小。

    >[AZURE.NOTE] 如果您展開在本機固定磁碟區，然後展開另一部在本機固定磁碟區，立即磁碟區擴充作業之後，依序執行。 必須先完成第一個磁碟區擴充作業，才能開始下一個磁碟區擴充作業。

![可用的視訊](./media/storsimple-manage-volumes-u2/Video_icon.png) **可用的視訊**

若要看影片，示範如何擴充磁碟區，按一下 [ [這裡](http://azure.microsoft.com/documentation/videos/expand-a-storsimple-volume)。

## 變更磁碟區類型

您可以將磁碟區類型從分層變更為固定在本機或從固定在本機變更為分層。 通常，這些磁碟區是您想要經常存取的小的現有磁碟區。 不過，這種轉換不應該經常發生。 將磁碟區從分層轉換為固定在本機的部分原因為：

- 關於資料可用性和效能的本機保證
- 消除雲端延遲和雲端連線問題。

當固定在本機的磁碟區建立時，它就會完整佈建。 如果您要將分層磁碟區轉換為固定在本機的磁碟區，StorSimple 會先確認您在裝置上有足夠的空間，再開始轉換。 如果您沒有足夠的空間，您會收到錯誤，且作業將會取消。 

> [AZURE.NOTE] 在開始從轉換分層至本機固定之前，請確定您考慮其他工作負載的空間需求。 

如果您需要額外空間來佈建其他磁碟區，您可能想要將固定在本機的磁碟區變更為分層磁碟區。 當您將固定在本機的磁碟區轉換為分層時，裝置上的可用容量會隨著釋放的容量大小增加。 如果連線問題讓磁碟區無法從本機類型轉換為分層類型，本機磁碟區會展示分層磁碟區的屬性，直到完成轉換為止。 這是因為可能有部分資料溢出到雲端。 此溢出的資料將繼續佔用裝置上的本機空間，無法釋放，直到重新啟動並完成作業。

>[AZURE.NOTE] 轉換磁碟區可能需要一些時間，並在啟動之後，您無法取消轉換。 磁碟區在轉換時仍然保持連線，您可以建立備份，但是您無法在轉換正在進行時擴充或還原磁碟區。  

#### 變更磁碟區類型

1. 在 **裝置** 頁面中，選取裝置，並按兩下，然後按一下 **磁碟區容器** ] 索引標籤。

2. 從清單中選取磁碟區容器，然後按兩下它以檢視與該容器相關聯的磁碟區。

3. 選取磁碟區，然後在頁面底部，按一下 [ **修改**。 [修改磁碟區精靈] 隨即啟動。

4. 在 **基本設定** 頁面上，選取新的類型從變更的使用類型 **使用類型** 下拉式清單中。

    - 如果您要變更的型別 **在本機固定**, ，StorSimple 會檢查是否有足夠的容量。
    - 如果您要變更的型別 **階層** 和封存資料，選取適用於此磁碟區 **用於較不常存取的封存資料的這個磁碟區** 核取方塊。

    ![封存核取方塊](./media/storsimple-manage-volumes-u2/ModifyVolume1.png)

5. 按一下箭頭圖示 ![箭號圖示](./media/storsimple-manage-volumes-u2/HCS_ArrowIcon.png) 前往 **其他設定** 頁面。

    下列訊息隨即出現。

    ![變更磁碟區類型訊息](./media/storsimple-manage-volumes-u2/TypeChangeMessage.png)

6. 按一下箭號圖示 ![箭號圖示](./media/storsimple-manage-volumes-u2/HCS_ArrowIcon.png) 繼續進行。

7. 按一下核取圖示 ![核取圖示](./media/storsimple-manage-volumes-u2/HCS_CheckIcon.png) 若要開始轉換程序。 Azure 入口網站將會顯示更新磁碟區訊息。 如果磁碟區已成功更新，即會顯示成功訊息。

## 使磁碟區離線

您想要修改或刪除磁碟區時，可能需要先使其離線。 當磁碟區離線時，即無法進行讀寫存取。 您必須使主機與裝置上的磁碟區同時離線。 

#### 若要使磁碟區離線

1. 請確定有問題的磁碟區不在使用中後，再使其離線。

2. 先使主機上的磁碟區離線。 這樣做可以排除任何會造成磁碟區上資料損毀的潛在風險。 如需特定步驟，請參閱主機作業系統的指示。

3. 在主機離線之後，請執行下列步驟以使裝置上的磁碟區離線：

  1. 在 **裝置** 頁面中，選取裝置，並按兩下，然後按一下 **磁碟區容器** ] 索引標籤。  **磁碟區容器** 索引標籤會列出以表格格式與裝置相關聯的所有磁碟區容器。
  2. 選取並按一下磁碟區容器，以清單顯示該容器內的所有磁碟區。
  3. 選取磁碟區，然後按一下 **離線**。
  4. 當系統提示您確認，按一下 [ **是**。 磁碟區現在應已離線。

    After a volume is offline, the **Bring Online** option becomes available.

> [AZURE.NOTE]  **離線** 命令會將要求傳送到裝置，以使磁碟區離線。 如果主機仍在使用該磁碟區，這會導致連線中斷，但是使磁碟區離線不會失敗。 

## 刪除磁碟區

> [AZURE.IMPORTANT] 您只能刪除磁碟區離線。

請完成下列步驟來刪除磁碟區。

#### 若要刪除磁碟區

1. 在 **裝置** 頁面中，選取裝置，並按兩下，然後按一下 **磁碟區容器** ] 索引標籤。

2. 選取含有您想要刪除之磁碟區的磁碟區容器。 按一下該磁碟區容器來存取 **磁碟區** 頁面。

3. 與這個容器相關聯的所有磁碟區會以表格顯示。 檢查您想要刪除之磁碟區的狀態。 如果您想要刪除的磁碟區未離線，使其離線首先，依照步驟 [使磁碟區離線](#take-a-volume-offline)。

4. 磁碟區離線後，按一下 [ **刪除** 頁面的底部。

5. 當系統提示您確認，按一下 [ **是**。 此時會刪除磁碟區和 **磁碟區** 頁面會顯示磁碟區容器內的更新的清單。

    >[AZURE.NOTE] 如果您刪除本機固定磁碟區，然後再刪除另一部在本機固定磁碟區立即之後，工作依序執行的磁碟區刪除。 必須先完成第一個磁碟區刪除作業，才能開始下一個磁碟區刪除作業。
 
## 監視磁碟區

監視磁碟區可讓您為磁碟區收集 I/O 相關的統計資料。 根據預設，您建立的前 32 個磁碟區會啟用監視。 預設會停用監視其他磁碟區。 預設也會停用監視複製的磁碟區。

請執行下列步驟來啟用或停用監視磁碟區。

#### 若要啟用或停用監視磁碟區

1. 在 **裝置** 頁面中，選取裝置，並按兩下，然後按一下 **磁碟區容器** ] 索引標籤。

2. 選取磁碟區所在的容器所在的磁碟區，然後再按一下該磁碟區容器來存取 **磁碟區** 頁面。

3. 以表格顯示所列出與這個容器相關聯的所有磁碟區。 按一下並選取磁碟區或磁碟區複製。

4. 在頁面底部，按一下 [ **修改**。

5. 在 [修改磁碟區精靈] 在 **基本設定**, ，請選取 **啟用** 或 **停用** 從 **監視** 下拉式清單中。
