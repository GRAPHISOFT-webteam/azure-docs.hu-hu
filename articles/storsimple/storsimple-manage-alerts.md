# 使用 StorSimple Manager 服務來檢視和管理 StorSimple 警示

## 概觀

 **警示** StorSimple Manager 服務中的索引標籤可讓您檢閱和清除即時的 StorSimple 裝置相關警示。 從這個索引標籤中，您可以集中監視 StorSimple 裝置和整個 Microsoft Azure StorSimple 解決方案的健康問題。

本教學課程說明常見的警示狀況、警示嚴重性層級及如何設定警示通知。 此外，也包含警示快速參考表，可讓您快速找出特定的警示並適當地回應。

![警示頁面](./media/storsimple-manage-alerts/HCS_AlertsPage.png)

## 常見的警示狀況

StorSimple 裝置會產生警示以回應各種不同的狀況。 以下是最常見的警示狀況類型：

- **硬體問題** – 這些警示將告訴您有關您硬體的健全狀況。 它們讓您知道是否需要升級韌體、網路介面是否有問題，或您的其中一個資料磁碟機是否有問題。

- **連線問題** – 將資料傳輸的困難度時，會出現這些警示。 在往返於 Azure 儲存體帳戶傳輸資料期間，或因為裝置與 StorSimple Manager 服務之間沒有連線時，可能會發生通訊問題。 通訊問題最難解決，因為失敗點實在太多。 在繼續進行更進階的疑難排解之前，您一定要先確認有網路連線和網際網路存取可用。 如需疑難排解的協助，請移至 [測試連接 cmdlet 疑難排解](storsimple-troubleshoot-deployment.md/#troubleshoot-with-the-test-hcsmconnection-cmdlet)。

- **效能問題** – 當您的系統不理想的情況下，例如時的重負載下執行時，會導致這些警示。

此外，您也可能會看到有關安全性、更新或作業失敗的警示。

## 警示嚴重性層級

視警示狀況造成的影響和是否需要回應警示而定，警示有不同的嚴重性層級。 嚴重性層級如下：

- **重大** – 此警示是回應的條件，會影響系統的成功的效能。 需要採取動作以確保 StorSimple 服務不中斷。

- **警告** – 這種情況可能會演變成重大如果無法解決。 您應該調查這種情況，並採取任何必要的動作來解決問題。

- **資訊** – 此警示包含可用於追蹤和管理系統的資訊。

## 設定警示設定

您可以選擇是否要透過電子郵件收到每個 StorSimple 裝置的警示狀況通知。 此外，您可以識別其他警示通知收件者輸入其電子郵件地址 **其他電子郵件收件者** ] 方塊中，並以分號隔開。

>[AZURE.NOTE] 您可以輸入最多 20 個電子郵件地址每個裝置。

啟用裝置的電子郵件通知之後，每當發生重大警示時，通知清單的成員將會收到電子郵件訊息。 將訊息傳送從 *storsimple-alerts-noreply@mail.windowsazure.com* 並說明警示狀況。 收件者可以按一下 **取消訂閱** 將自己從電子郵件通知清單中移除。

#### 啟用裝置的警示電子郵件通知

1. 移至 **裝置** > **設定** 裝置。

2. 在 **警示設定**, ，進行下列設定 ︰

    1. 在 **傳送電子郵件通知** 欄位中，選取 **是**。

    2. 在 **電子郵件服務系統管理員** 欄位中，選取 **是** 如果您想要讓服務管理員和所有共同管理員收到警示通知。

    3. 在 **其他電子郵件收件者** 欄位中，輸入所有其他收件者應該會收到警示通知的電子郵件地址。 輸入名稱，格式 *someone@somewhere.com*。 使用分號來分隔電子郵件地址。 您可以對每一裝置設定最多 20 個電子郵件地址。 

        ![警示通知設定](./media/storsimple-manage-alerts/AlertNotify.png)

3. 若要傳送測試電子郵件通知，請按一下箭號圖示旁邊 **傳送測試電子郵件**。 StorSimple Manager 服務轉寄測試通知時會顯示狀態訊息。 

4. 下列訊息出現時，按一下 [ **確定**。 

    ![已傳送警示測試通知電子郵件](./media/storsimple-manage-alerts/HCS_AlertNotificationConfig3.png)

    >[AZURE.NOTE] 如果無法傳送測試通知訊息，StorSimple Manager 服務將會顯示適當的訊息。 按一下 [ **確定**, ，稍候數分鐘，然後嘗試再次傳送測試通知訊息。 

## 檢視和追蹤警示

StorSimple Manager 服務儀表板可讓您快速概覽裝置上的警示數目 (依嚴重性層級排列)。

![警示儀表板](./media/storsimple-manage-alerts/admin_alerts_dashboard.png)

按一下嚴重性層級會開啟 **警示** ] 索引標籤。 結果只包含符合該嚴重性層級的警示。

![限定警示類型的警示報告](./media/storsimple-manage-alerts/admin_alerts_scoped.png)

按一下清單中的警示會提供警示的其他詳細資料，包括上次報告警示的時間、裝置上發生該警示的次數，以及建議採取來解決警示的動作。 如果是硬體警示，則也會識別硬體元件。

![硬體警示範例](./media/storsimple-manage-alerts/admin_alerts_hardware.png)

如果您需要將資訊傳送給 Microsoft 支援服務，您可以將警示詳細資料複製到文字檔案。 您已經遵循建議及解決警示條件內部之後，您應該選取中的警示來清除裝置警示 **警示** ] 索引標籤，然後按一下 **清除**。 若要清除多個警示，按下 Ctrl 鍵同時選取警示，然後按一下 [ **清除**。 請注意，當問題解決時或系統以新資訊更新警示時，會自動清除某些警示。

當您按一下 **清除**, ，您將有機會提供警示，您用來解決此問題的步驟的相關註解。 如果新的資訊觸發另一個事件，系統會清除某些事件。 在此情況下，您會看到下列訊息。

![清除警示訊息](./media/storsimple-manage-alerts/admin_alerts_system_clear.png)

## 排序和檢閱警示

執行警示報告以分組檢閱和清除警示可能更有效率。 此外， **警示** ] 索引標籤可以顯示 250 個最多警示。 如果超過該警示數目，則並非所有警示都會出現在預設檢視中。 您可以結合下列欄位來自訂要顯示的通知：

- **狀態** – 您可以顯示 [ **Active** 或 **已清除** 警示。 作用中警示仍在系統上觸發，而已清除的警示已由系統管理員手動清除，或因為系統以新資訊更新警示狀況而以程式設計方式清除。

- **嚴重性** – 您可以顯示所有嚴重性層級 （重大、 警告、 資訊） 或只是特定嚴重性的警示，例如只有重大警示的警示。

- **來源** – 您可以顯示來自所有來源，或只為來自服務或其中一個或所有裝置的警示。

- **時間範圍** – 藉由指定 **從** 和 **至** 日期和時間戳記，您可以查看警示您感興趣的時段內。

### 雲端連線失敗時的 StorSimple 行為

若雲端連線失敗時，在生產環境中執行的 StorSimple 裝置會發生什麼情況？

如果 StorSimple 生產裝置的雲端連線失敗，視您的裝置狀態，可能會發生下列狀況： 

- **針對您的裝置上的本機資料**︰ 段時間，將會不會中斷和讀取作業將繼續提供服務。 不過，當未完成的 IO 數量增加並超過限制時，讀取可能就會失敗。 

    視裝置上的資料量，雲端連線中斷後的數小時內仍會持續寫入。 寫入速度會逐漸緩慢，最終會於雲端連線中斷數小時後失敗。 (裝置上有即將推送至雲端之資料的暫存儲存體。 傳送資料時，這個區域會排清。 如果連線失敗，此儲存區域中的資料不會推送到雲端，而且 IO 會失敗。)   

 
- **在雲端中的資料**︰ 大部分的定域機組連線錯誤，則會傳回錯誤。 只要連線還原時，IO 就會繼續進行，使用者無需自行連線磁碟區。 但在罕見情況下，有可能會需要使用者介入，透過 Azure 傳統入口網站連線磁碟區。 
 
- **在進行中的雲端快照集**: 4-5 個小時內多次重試此作業，而且無法恢復連線時，如果雲端快照集將會失敗。
