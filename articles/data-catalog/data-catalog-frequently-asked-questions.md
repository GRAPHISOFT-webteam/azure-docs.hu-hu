# Azure 資料目錄的常見問題集

本文提供 microsoft 相關的常見問題的解答 **Azure 資料目錄** 服務。

## 問 ︰ 什麼是 **Azure 資料目錄**？

答 ︰ Microsoft **Azure 資料目錄** 是完全受管理的服務裝載於 Microsoft Azure 雲端，可作為的註冊系統和探索企業資料來源系統。 **Azure 資料目錄** 提供功能，可讓任何使用者 – 從分析師資料科學家到開發人員 – 註冊、 探索、 了解及取用資料來源。

## 問：Azure 資料目錄可解決客戶面臨的什麼挑戰？

**Azure 資料目錄** 可讓使用者探索和了解企業資料來源，以解決資料來源探索和 「 黑暗資料 」 的挑戰。

## 問：Azure 資料目錄的目標對象是誰？

**Azure 資料目錄** 提供功能的技術和非技術使用者，包括 ︰

- 資料開發人員、BI 和分析專業人員：負責產生資料和分析內容供他人使用
-   資料負責人：這些人了解資料、其意義、用法和用途
- 資料取用者：這些人需要能夠透過他們所選擇的工具，輕鬆地尋找、了解並連線至工作所需的資料
- 中央 IT：這些人需要提供數百個資料來源供商務使用者探索，也需要持續監督資料的使用情況和由誰使用

## 問：Azure 資料目錄的區域可用性如何？

在預覽期間， **Azure 資料目錄** 都可使用服務只有下列資料中心 ︰

- 美國西部
- 美國東部
- 歐洲西部
- 澳洲東部

## 問：Azure 資料目錄中的資料資產數目有何限制？

免費版的 **Azure 資料目錄** 僅限於 5000 已註冊的資料資產。

標準版的 **Azure 資料目錄** 支援多達 100000 個已註冊的資料資產。

## 問：支援什麼資料來源和資產類型？

在預覽期間， **Azure 資料目錄** 目前支援的 SQL Server 關聯式 (包括 Azure SQL DB) 和 SQL Server Analysis Services （多維度和表格式） 資料庫，以及 SQL Server Reporting Services （僅限原生模式） 和 Oracle 資料庫。

在預覽期間， **Azure 資料目錄** 支援註冊與探索下列資產類型 ︰

- SQL Server 資料表
- SQL Server 檢視
- SQL Server Analysis Services 維度
- SQL Server Analysis Services 量值
- SQL Server Analysis Services KPI
- SQL Server Analysis Services 資料表
- SQL Server Reporting Services 報表  
- Oracle 資料庫資料表
- Oracle 資料庫檢視
- Azure 儲存體 Blob
- Azure 儲存體目錄
- HDFS 檔案
- HDFS 目錄
- Teradata 資料表
- Teradata 檢視

除了以上列出的資產 **Azure 資料目錄** 支援下列資料資產容器探索 ︰

- SQL Server Database
- Oracle 資料庫
- HDFS 叢集
- Teradata 資料庫

## 問：如何開始使用 Azure 資料目錄？

若要開始的最佳位置是在指示 [開始使用資料目錄](../data-catalog-get-started/)。 本文是公開預覽中各項功能的端對端教學課程。

## 問：如何註冊我的資料？

若要註冊您的資料 **Azure 資料目錄**, ，啟動 **Azure 資料目錄** 註冊工具的 [發佈] 區 **Azure 資料目錄** 入口網站。 在 **Azure 資料目錄** 發行應用程式，使用登入您所使用的相同認證存取 **Azure 資料目錄** 入口網站，然後選取資料來源和您想要註冊的特定資產。

## 問：從已註冊的資料資產中會擷取哪些屬性？

資料來源，但是在一般資料來源不同的特定屬性 **Azure 資料目錄** 發佈服務會擷取下列資訊 ︰

- 資產名稱
- 資產類型
- 資產描述
- 屬性/資料行名稱
- 屬性/資料行資料類型
- 屬性/資料行描述

> [AZURE.IMPORTANT] 註冊資料資產 **Azure 資料目錄** 不會移動或複製資料到雲端。 從資料來源註冊資產會將資產的中繼資料複製到 Azure，但資料仍保留在現有的資料來源位置。 此規則唯一的例外狀況，在於使用者註冊資產時選擇上傳預覽記錄或資料設定檔。 當包含預覽，最多 20 筆記錄會複製從每個資產，並做為快照集儲存 **Azure 資料目錄**。 包含資料設定檔時，系統將會計算彙總資訊 (例如資料表大小、每資料行 null 值百分比，以及各資料行的最小值、最大值和平均值)，並將其加入目錄所存放的中繼資料。

<br/>

> [AZURE.NOTE] 如有第一級的 SQL Server Analysis Services 資料來源的 **描述** 屬性， **Azure 資料目錄** 發佈應用程式會擷取該屬性值。 對於 SQL Server 關聯式資料庫，因為缺少第一級 **描述** 屬性， **Azure 資料目錄** 發行應用程式將會從 ms_description 擴充屬性的物件和資料行擷取值。 如需詳細資訊，請參閱 TechNet [使用擴充屬性加入資料庫物件上](https://technet.microsoft.com/library/ms190243%28v=sql.105%29.aspx)。

## 問：新註冊的資產經過多久才會出現在 Azure 資料目錄中？

註冊資產之後 **Azure 資料目錄** 可能會有 5-10 秒才會出現在一段 **Azure 資料目錄** 入口網站。

## 問：如何加註和充實已註冊的資料資產的中繼資料？

已註冊的資產提供中繼資料最簡單的方法是選取資產 **Azure 資料目錄** 入口網站，然後輸入中繼資料值在屬性] 窗格或結構描述] 窗格中選取的物件。

在註冊過程中，您也可以提供一些中繼資料，例如專家和標記。 中提供的值 **Azure 資料目錄** 發行服務將會套用到當時註冊的所有資產。 若要檢視最近註冊的物件，額外的註釋的入口網站中，選取 **檢視入口網站** 的最後一個畫面上的按鈕 **Azure 資料目錄** 發行應用程式。

## 問：如何刪除我已註冊的資料物件？

您可以刪除的物件從 **Azure 資料目錄** 入口網站中，在選取的物件，然後按一下 **刪除** ] 按鈕。 這會移除從物件的中繼資料 **Azure 資料目錄** 但不是會影響實際的基礎資料來源。

## 問：什麼是專家？

專家是指對資料物件具有獨特見解的人。 一個物件可能有多個專家。 專家不見得是物件的「擁有者」。專家只是知道應該如何運用資料的人。

## 問：什麼是預覽的 SLA？

在 **Azure 資料目錄** 預覽，沒有明確的服務等級協議。

## 問：如果我遇到問題，如何與 Azure 資料目錄小組分享資訊？

請使用 **Azure 資料目錄** 報告問題、 分享資訊及發問的論壇。 此論壇位於 http://go.microsoft.com/fwlink/?LinkID=616424&clcid=0x409

##問：Azure 資料目錄可以處理我想要使用的其他資料來源嗎？
我們正積極新增更多的資料來源，以 **Azure 資料目錄**。 在您想要查看支援，請建議 （或支持如果已建議） 的資料來源是否 [Azure 資料目錄論壇](http://go.microsoft.com/fwlink/?LinkID=616424&clcid=0x409)。

## 問：Azure 資料目錄與 Power BI for Office 365 中的資料目錄有何關係？

您可以將 **Azure 資料目錄** 做為資料目錄的進化。 **Azure 資料目錄** 提供類似的功能，資料來源發行和探索，但會著重於更廣泛的案例並不依賴 Office 365。 不久之後 **Azure 資料目錄** 會變成可供使用兩個類別目錄將會合併到單一服務。

## 問：使用者向 Azure 資料目錄註冊資產需要什麼權限？

使用者執行 **Azure 資料目錄** 註冊工具需要的權限允許讀取中繼資料來源的資料來源。 如果使用者也選擇包含預覽，則使用者必須也有權限從註冊的物件讀取資料。

## 問：Azure 資料目錄也會支援內部部署嗎？

**Azure 資料目錄** 是雲端服務可以處理的雲端和內部部署資料來源，提供混合式資料來源探索解決方案。 目前也不打算使用之版本的 **Azure 資料目錄** 會在內部執行的服務。

##問：我們是否可以從我們所註冊的資料來源中擷取更多/更豐富的中繼資料？

我們正積極擴充功能的 **Azure 資料目錄**。 如果您想要查看擷取從資料來源在註冊期間的其他中繼資料，請提出建議 （如果已建議投票） 在 [Azure 資料目錄論壇](http://go.microsoft.com/fwlink/?LinkID=616424&clcid=0x409)。 未來，我們將允許協力廠商透過擴充性 API 加入新的資料來源類型。

## 問：如何限制已註冊的資料資產的可見性，以限制只有特定人員才能探索它們？

答 ︰ 選取的資料資產中 **Azure 資料目錄**, ，然後按一下 [取得擁有權 」 按鈕。 中的資料資產的擁有者 **Azure 資料目錄** 可以變更可見性設定為允許所有目錄使用者探索擁有的資產，或限制特定使用者的可見性。

## 問：如何更新資料資產的註冊，讓資料來源中的變更反映在目錄中？

答：若要更新已註冊在目錄中的資料資產的中繼資料，只要重新註冊包含這些資產的資料來源即可。 資料來源中的任何變更 (例如在資料表或檢視中加入或移除資料行) 都會在目錄中更新，但會保留使用者提供的任何註解。

如果您遇到問題，或需要協助 **Azure 資料目錄** 預覽，請張貼在 [Azure 資料目錄論壇](http://go.microsoft.com/fwlink/?LinkID=616424&clcid=0x409)。
