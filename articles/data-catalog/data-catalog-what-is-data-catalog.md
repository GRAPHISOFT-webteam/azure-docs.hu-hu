# 什麼是 Azure 資料目錄？

Microsoft **Azure 資料目錄** 是完全受管理的雲端服務，可作為的註冊系統和探索企業資料來源系統。 **Azure 資料目錄** 提供功能，可讓任何使用者 – 從分析師，若要建置和支援社群和資料的文化特性的資料科學家到開發人員 – 探索、 了解，以及使用資料來源，並提供他們的知識。

## 問題描述 - 動機和概觀

傳統上，探索企業資料來源一向都是根據部落知識的一項有機程序。 對於想要充分利用其資訊資產的公司，這帶來許多挑戰。

-   除非使用者在另一個程序中接觸到資料來源，否則並不知道有此資料來源。沒有一個可註冊資料來源的中央位置。
-   除非使用者知道資料來源的位置，否則無法利用用戶端應用程式連接到資料。使用者必須知道連接字串或路徑，才能取用資料。
-   除非使用者知道資料來源的文件位置，否則無法了解資料的用途。資料來源和文件位於不同的地方，需要透過不同的方式才能取得。
-   如果使用者對資訊資產有疑問，他必須找到負責資料的專家或小組，並在離線狀態下請教這些專家。在資料與資料用途上具備專業觀點的專家之間，未建立明確的銜接管道。
-  除非使用者了解要求存取資料來源的程序，否則，探索資料來源及其文件仍無法讓他存取所需的資料。

雖然資料取用者面臨這些挑戰，但負責產生和維護的資訊資產的使用者本身也面臨挑戰。

-   使用描述性中繼資料加註資料來源往往白費心力。用戶端應用程式通常會忽略儲存在資料來源中的描述。
-   建立資料來源的文件通常是白費心力。保持文件與資料來源同步是一種持續性的責任，使用者總是認為文件過時，所以都不信任文件。
- 限制對資料來源的存取，並確保資料取用者知道如何要求存取是一項持續的挑戰。

建立和維護資料來源的文件既複雜又耗時。 為了讓使用資料來源的每個使用者可隨時取得文件，所面臨的挑戰通常更艱鉅。

這些挑戰結合起來形成更大的障礙，使得公司難以鼓勵和推動使用和了解企業資料。

## 服務描述

**Azure 資料目錄** 設計來解決這些問題，讓企業能夠讓它們輕鬆地探索和了解使用者在需要管理的資料時，從其現有的資訊資產，取得最大價值。

**Azure 資料目錄** 提供哪些資料，可以註冊來源的雲端的服務。 資料會保留在現有的位置，但一份中繼資料，以及資料來源位置的參考，則加入至 **Azure 資料目錄**。 此中繼資料也會編製索引，能夠透過搜尋輕鬆找到每個資料來源，並讓探索資料來源的使用者了解。

註冊資料來源之後，即可由執行註冊的使用者或企業中的其他使用者來充實其中繼資料。 任何使用者都可以提供描述、標記或其他中繼資料 (例如要求資料來源存取的文件和程序) 來加註資料來源。 此描述性中繼資料可補充資料來源中已註冊的結構化中繼資料 (例如資料行名稱和資料類型)，讓使用者更輕鬆地探索和了解。

註冊來源的主要目的是為了探索和了解資料來源及其用途。 當企業使用者需要的大力 （這可能是商業智慧、 應用程式開發、 資料科學或任何其他工作需要正確資料的） 資料時可以使用 **Azure 資料目錄** 的探索體驗，快速尋找資料符合他們的需求，了解要評估其適合用途，並使用該資料在其選擇的工具中開啟資料來源的資料。 在此同時， **Azure 資料目錄** 可讓使用者至目錄，標記的方式，提供記錄，且加註已註冊，並註冊新的資料來源的資料來源可以然後探索、 了解，並由目錄使用者社群。

## 註冊資料來源

資料來源註冊透過 **Azure 資料目錄** 資料來源註冊工具。 此應用程式可以從下載 **Azure 資料目錄** 入口網站。

註冊程序包含三個基本步驟：

1.  連接到資料來源 - 使用者指定資料來源位置和認證，以連接到資料來源，例如 SQL Server 執行個體。
2.  選取註冊-使用者選取的物件的物件在指定的位置應該向 **Azure 資料目錄**。 這可能是伺服器上所有資料庫中的全部資料表，或特別選取的一組資料表和檢視。
3.  完成註冊-使用者完成此程序，與資料來源註冊工具資料來源擷取結構化中繼資料，並傳送該中繼資料來 **Azure 資料目錄** 雲端服務。

> [AZURE.NOTE] 預覽版本中， **Azure 資料目錄** 目前支援下列資料來源和資產類型 ︰

- SQL Server 資料表
- SQL Server 檢視
- Oracle 資料庫資料表
- Oracle 資料庫檢視
- Teradata 資料表
- Teradata 檢視
- SQL Server Analysis Services 多維度的維度
- SQL Server Analysis Services 多維度的量值
- SQL Server Analysis Services 多維度的 KPI
- SQL Server Analysis Services 表格式資料表
- SQL Server Reporting Services 報表
- Azure 儲存體 Blob
- Azure 儲存體目錄
- HDFS 檔案
- HDFS 目錄
- Hive 資料表

其他資料來源和資產類型會新增 **Azure 資料目錄** 預覽。

> [AZURE.IMPORTANT]登錄中的資料來源 **Azure 資料目錄** 不會複製資料來源的資料，除非您選取 [包含預覽] 中的資料來源註冊工具。 註冊會複製資料來源中繼資料，而非資料。 中繼資料的例子包括資料表和其他資料來源物件的名稱，還有資料行的名稱和資料類型及其他資料來源屬性。 中繼資料也包含資料來源位置，讓使用者在探索資料來源使用 **Azure 資料目錄** 可以連接到資料來源。 如果您選取 [包含預覽]，則資料來源註冊工具也會將複製到 **Azure 資料目錄** 將探索資料來源中的使用者顯示的記錄一小群 **Azure 資料目錄** 入口網站。

## 充實資料來源中繼資料

註冊完成之後，可以探索並取用，則為 true 的值以外的資料來源 **Azure 資料目錄** 來自具有描述性商務中繼資料中擷取資料來源的結構化中繼資料一樣。  此額外的中繼資料帶來三個重要的優點：

-   更輕易地找到已註冊的資料來源。 使用者提供的中繼資料加入至 **Azure 資料目錄** 搜尋索引。 這可讓使用者利用原始資料來源中可能沒有的詞彙和概念來探索資料。 比方說，如果一個包含客戶資料的資料庫資料表名為 "tbl_c45"，則提供易記名稱 "Customer"，將可讓尋找客戶資料的使用者更輕易找到此資料表。 同樣地，如果提供的描述包含用到資料的報表、儀表板或程序的名稱，將可讓使用這些下游構件作為搜尋詞彙的使用者，更容易找到資料來源。
-   一旦找到已註冊的資料來源，也能更容易了解。 使用者提供的中繼資料會顯示任何 **Azure 資料目錄** 使用者在檢視加註的資料來源，這有助於提供額外的脈絡和資訊。 大部分資料來源通常不包含有意義的描述或文件，而有這樣做的資源來源通常是以技術性 DBA 或資料庫開發人員為對象。 來充實資料來源中的 **Azure 資料目錄** 與適當的對象的描述和標記的使用者可以協助確保探索資料的任何人能夠了解其詳細資料，而且能使用。
-  每個已註冊的資料來源都可以包括要求存取資訊，讓使用者可以輕鬆地了解並遵循現有的程序，要求對資料來源及其資料的存取。

> [AZURE.NOTE] 每個 **Azure 資料目錄** 使用者可以加入自己的標記和資料資產和屬性的描述。 **Azure 資料目錄** 會追蹤值和每個註解的來源，且會顯示加入它的使用者。 這種對於中繼資料的群眾外包作法，可確保對資料及其用途有其見解的每個使用者，能夠與整個使用者社群分享其觀點和資源。

## 瀏覽、探索及了解

目標的註冊和充實資料來源中 **Azure 資料目錄** ，讓他們能夠探索、 了解，並由整個企業的使用者。  **Azure 資料目錄** 入口網站是此程序的主要工具。

 **Azure 資料目錄** 入口網站提供兩種主要機制資料瀏覽及探索 ︰ 搜尋和篩選。

若要搜尋 **Azure 資料目錄** 對於資料來源，只要輸入搜尋詞彙 」 中的 [搜尋] 方塊中 **Azure 資料目錄** 入口網站。 入口網站會將符合搜尋詞彙的每個已註冊的資料來源顯示成一個磚。這些磚包含指派給資料來源的名稱、描述和標記，以及其他高階資訊。

若要篩選的內容 **Azure 資料目錄**, 、 只選取一個或多個篩選器中呈現 **Azure 資料目錄** 入口網站。 這樣可限制入口網站只顯示符合指定篩選準則的磚。 您可以不經過搜尋而篩選資料來源，也可以篩選搜尋結果。

若要檢視資料來源更完整的資訊，並了解是否適合手邊的工作，只要按一下資料來源的磚。將會顯示屬性窗格，其中包含其所有中繼資料。

屬性窗格的頂端會有其他按鈕：

1.  預覽：選取此按鈕會顯示資料來源中的一組靜態預覽記錄 (如果已在資料來源註冊期間選取預覽)。
2.  結構描述 ︰ 選取此按鈕會顯示資料來源，包括資料行名稱和資料型別，以及在任何資料行層級中繼資料的結構描述 **Azure 資料目錄**。

> [AZURE.NOTE] 請務必記得 **探索** 體驗可能是進入點 **擴充** 體驗，而不只是進入 **取用** 體驗。 群眾外包方法， **Azure 資料目錄** 帶來意謂著任何使用者在探索已註冊的資料來源可以分享他對於資料，除了使用他找到的資料的觀點。

## 移除資料來源中繼資料

註冊資料來源之後，有時可能需要移除資料來源參考，從 **Azure 資料目錄**。 這可能是因為變更商務需求或淘汰來源系統。 不論原因如何， **Azure 資料目錄** 輕鬆地移除資料來源，只要選取刪除，讓它們不再可探索和取用。

> [AZURE.IMPORTANT] 刪除資料來源從 **Azure 資料目錄** 只會刪除中繼資料儲存在 **Azure 資料目錄** 服務。 無論如何都不會影響原始資料來源。

## 使用資料來源

資料探索的最終目標是尋找您需要的資料，並透過您選擇的資料工具來使用它。 Azure 資料目錄中的資料取用體驗以兩種作法實現此目標。

1.  用戶端應用程式直接支援的 **Azure 資料目錄**, ，使用者可以按一下 **開啟於** ] 功能表中的資料來源磚在網站上找到。 用戶端應用程式接著就會啟動，並連接到選取的資料來源。
2.  對於所有用戶端應用程式，使用者可以使用選取的資料來源的屬性窗格中所顯示的連接資訊。 此資訊包含連接到資料所需的一切詳細資料 (例如伺服器名稱、資料庫名稱和物件名稱)，而且可以複製到用戶端工具的連線體驗中。 如果已針對資料來源提供要求存取詳細資料，這項資訊將會顯示在連線詳細資料的旁邊。

> [AZURE.NOTE] Azure 資料目錄預覽版本中，只有 Microsoft Excel 和 SQL Server Reporting Services 報表管理員會直接支援，而且可以在 **開啟於** 功能表。
