## 2014/8/15 之前所建立之叢集的附註

Azure PowerShell 或 HDInsight SDK 錯誤訊息，「 叢集 <clustername> 未設定 HTTP 服務存取 」 (或依作業的其他錯誤訊息，例如: 「 無法連線到叢集 」)，起因於 Azure PowerShell 或 HDInsight SDK 和叢集之間的版本差異。 在 8/15 或之後建立的叢集支援佈建到虛擬網路的這項新功能。 舊版的 Azure PowerShell 或 HDInsight SDK 無法正確解譯此功能，導致工作提交作業失敗。 如果您使用 HDInsight SDK API 或 Azure PowerShell Cmdlet (例如 Use-AzureRmHDInsightCluster 或 Invoke-AzureRmHDInsightHiveJob) 來提交工作，這些作業可能會失敗並傳回上述其中一個錯誤訊息。

最新版的 HDInsight SDK 和 Azure PowerShell 中已解決這些相容性問題。 建議將 HDInsight SDK 更新到 1.3.1.6 版或更新版本，並將 Azure PowerShell 工具更新到 0.8.8 版或更新版本。 您可以從最新的 HDInsight sdk 來取得存取 [NuGet][nuget-link]。 您可以使用，以存取 Azure PowerShell 工具 [Microsoft Web Platform Installer][webpi-link]。

HBase 可讓您在 HDInsight 上建置各種即時工作負載，包括處理大型資料集的互動式網站，甚至是將數百萬個端點的感應器和遙測資料儲存起來的服務，都沒問題。 下一步是使用 Hadoop 工作來分析這些工作負載中的資料，而透過 Azure PowerShell 和 Hive 叢集儀表板，就能在 HDInsight 中進行此分析。

### Apache Mahout 預先安裝在 HDInsight 3.1

 [Mahout](http://hortonworks.com/hadoop/mahout/) 已預先安裝在 HDInsight 3.1 Hadoop 叢集，因此您可以執行 Mahout 工作，而不需要其他叢集組態。 例如，您可以使用遠端桌面通訊協定 (RDP) 從遠端進入 Hadoop 叢集，且不需要額外的步驟就能執行下列 Hello World Mahout 命令：

        mahout org.apache.mahout.classifier.df.tools.Describe -p /user/hdp/glass.data -f /user/hdp/glass.info -d I 9 N L  

        mahout org.apache.mahout.classifier.df.BreimanExample -d /user/hdp/glass.data -ds /user/hdp/glass.info -i 10 -t 100

此程序更完整說明，請參閱文件 [Breiman 範例](https://mahout.apache.org/users/classification/breiman-example.html) Apache Mahout 網站上。


### Hive 查詢可在 HDinsight 3.1 中使用 Tez

Hive 0.13 可在 HDInsight 3.1 中使用，且能夠使用 Tez 來執行查詢，妥善運用可大幅提升效能。
依預設不會對 Hive 查詢啟用 Tez。 若要使用，您必須自行啟用。 您可以執行下列程式碼片段來啟用 Tez：

        set hive.execution.engine=tez;
        select sc_status, count(*), histogram_numeric(sc_bytes,5) from website_logs_orc_local group by sc_status;

Hortonworks 在標準效能評比中，已發表一份關於 Hive 查詢搭配 Tez 而提升效能的詳細數據。 如需詳細資訊，請參閱 [基準測試 Apache Hive 13 for Enterprise Hadoop](http://hortonworks.com/blog/benchmarking-apache-hive-13-enterprise-hadoop/)。

如需有關搭配 Tez 使用 Hive 的詳細資訊，請參閱 [Tez 上的 Hive](https://cwiki.apache.org/confluence/display/Hive/Hive+on+Tez)。

###叢集版本之間的注意事項

**與 HDInsight 3.1 叢集搭配使用的 Oozie 中繼存放區不回溯相容，HDInsight 2.1 叢集，且無法搭配此舊版本**。

與 HDInsight 3.1 叢集一起部署的自訂 Oozie metastore 資料庫，無法在 HDInsight 2.1 叢集上重複使用。 即使是源自 HDInsight 2.1 叢集的中繼存放區也是如此。 不支援這種情況下，因為 metastore 結構描述取得升級，使它不會再與 HDInsight 2.1 叢集所需的 metastore 相容，HDInsight 3.1 叢集，搭配使用時。 嘗試重複使用已在 HDInsight 3.1 叢集上使用過的 Oozie 中繼存放區，將會使 HDInsight 2.1 叢集變得無法使用。

**叢集之間無法共用 Oozie 中繼存放區。**

Oozie 中繼存放區會連接到特定叢集，且兩者無法在叢集之間共用。

###重大變更

**前置詞語法**:
只有 「 wasb: / /"HDInsight 3.1 和 3.0 叢集支援語法。 HDInsight 2.1 和 1.6 叢集支援舊的 "asv://" 語法，但在 HDInsight 3.1 或 3.0 叢集中不支援。 這表示，任何工作提交至 HDInsight 3.1 或 3.0 叢集且明確使用"asv://: / / 」 語法將會失敗。 應該改用 "wasb://" 語法。 此外，如果工作提交至任何以現有中繼存放區建立的 HDInsight 3.1 或 3.0 叢集，且中繼存放區中使用 asv:// 語法來明確參考資源，則也會失敗。 必須使用 wasb:// 來定址資源以重新建立這些中繼存放區。


**連接埠**: HDInsight 服務所使用的連接埠已變更。 以前所使用的連接埠號碼都在 Windows 作業系統暫時連接埠範圍內。 對於短期的網際網路通訊協定通訊，會自動從預設定義的暫時範圍中配置連接埠。 新的一組允許的 Hortonworks Data Platform (HDP) 服務連接埠號碼不在此範圍內，以避免與前端節點上執行的服務所使用的連接埠發生衝突。 新的連接埠號碼應該不會引起任何重大變更。 使用的號碼如下：
